Regeln

Alle Lösungen müssen sich an die folgenden grundlegenden Regeln halten.

  • Formatierung nach den Standardregeln von Eclipse
  • englische Bezeichner
  • Java-typische Groß- und Kleinschreibung
  • keine Warnungen, keine Fehler
  • keine toter Code
  • sinnvolle Variablennamen
  • keine Methodenparameter verändern
  • keine Veränderung einer Variable, wenn nicht notwendig
  • kein break, kein return, um eine Schleife zu beenden
  • den Zähler nicht verändern, um eine Schleife zu beenden
