Anforderungen bei einer Abschlussarbeit/einem Projekt

Last changed on January 29, 2019

Auf dieser Seite werden grundlegende Anforderungen bei der Durchführung einer Abschlussarbeit/eines Projektes aufgelistet.

Selbst-Organisation

Der Quelltext, den Sie entwickeln, sollte mit einem Git-Repository versioniert werden. Sie können dazu ein privates oder öffentliches Repository bei GitHub nutzen oder auch ein Repository auf dem GitLab-Server des Fachbereichs nutzen.

Alle Arbeiten, die Sie durchführen, sollten in einem Ticket beschrieben sein. Dabei sind die Standardregeln bei der Beschreibung von Arbeitspaketen zu berücksichtigen, etwa die Prüfbarkeit und die Kleinteiligkeit.

Beim Schreiben von Commit-Nachrichten sollten diese Regeln beachtet werden. Außerdem sollte am Ende der Commit-Nachricht die Nummer des Tickets stehen, zu dem der Commit gehört. Auf diese Weise werden die Commits im Ticket referenziert.

Code-Qualität

Nutzen Sie bitte Tools, um eine hohe Code-Qualität zu erzielen. Wenn Sie in Ihrer Abschlussarbeit mit Haskell arbeiten, würde ich Ihnen zum Beispiel den Linter hlint und den Auto-Formater hindent empfehlen.

Ausarbeitung

Bitte achten Sie bei Ihrer Ausarbeitung auf die deutsche Rechtschreibung. Beachten Sie zum Beispiel das Konzept des Deppenleerzeichens, die Kommasetzung der deutschen Rechtschreibung, sowie die Unterscheidung zwischen das und dass.

LaTeX

Wenn Sie für Ihre Ausarbeitung LaTeX nutzen, sollten Sie die folgenden Punkte beachten.

  • Nutzen Sie ein entsprechende Paket zur Darstellung von Quelltext. Das Paket minted eignet sich hierfür gut, es benötigt allerdings ein entsprechendes Python-Paket. Ohne Python kommen Sie zum Beispiel mit listings aus.

Wenn Abbildungen in einer Ausarbeitung vorkommen, müssen diese immer erläutert werden.
